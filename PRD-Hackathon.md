# EchoRank / PeopleRank 黑客松 PRD

## 1. 名字与一句话
- 推荐英文名：EchoRank（用“人们的声音”给活动做可信背书的声誉排名）
- 推荐中文名：声誉回声 / 回声排名
- 备选：PeopleRank、VoxRank、ProofPulse、RepuVoice、SignalRank、CivicRank、CredCast、AttestRank
- 一句话：把每次活动的“口碑”变成可验证、可组合、可激励的链上声誉

## 2. 产品愿景
- 让“发 NFT/SBT/积分”不再流于形式，把活动/贡献的真实价值通过“足够多人的可验证反馈”固化为可用的声誉资产
- 面向数字经济与社区协作，建立跨社区可迁移的信誉度量与激励闭环

## 3. 背景与问题
- 痛点
  - 活动纪念 NFT/SBT 泛滥：只证明“来过”，不证明“有价值”
  - 机构信用参差：同样的 NFT，不同机构含金量差别巨大
  - 缺乏跨社区通用的信誉度量，协作与赞助难以基于“真实影响力”
- 目标
  - 对活动本身形成“人评背书分”（PeopleRank）
  - 对机构形成“发证可信度”（Org Reputation）
  - 对个人形成“可迁移贡献声誉”（Global Reputation）

## 4. 用户与主要场景
- 用户
  - 主办方/机构：发布活动、发放参与凭证、建立品牌信誉
  - 参与者/志愿者：获得有含金量的声誉与潜在权益
  - 受众/旁听者：贡献反馈，提高评价广度
  - 赞助方/合作社区：以资金/权益换取触达与协作
- 场景
  - 活动结束后，每人完成 **3 分钟语音调研**（约 3–5 个问题）
  - AI 分析语音并输出结构化结果，系统聚合为 PeopleRank，绑定链上凭证

## 5. MVP 范围（黑客松落地）
- 活动创建
  - 主办方创建活动页：时间/地点/主题/主办方信息
  - 配置 3–5 个调研问题（满意度、是否达成目标、最有价值点、改进建议、是否推荐）
- 参与与凭证
  - 钱包或邮箱登录
  - 现场二维码签到（或活动码）作为“参与证明”的弱门槛
- 语音调研
  - 移动端录音提交（每题限时，总时长约 3 分钟）
- AI 分析（结构化输出）
  - 语音转文本（ASR）
  - 语义摘要（提炼主题与要点）
  - 情感/满意度分析（正负向与强度）
  - 质量/可疑度信号（过短、模板化、跑题等）
- PeopleRank 计算与展示
  - 输出：总分、维度分、置信度（样本数/分布）、Top insights
  - 展示：活动页仪表盘、机构页历史曲线、个人页声誉变化
- 链上绑定（MVP）
  - 上链摘要哈希/内容地址 + 评分结果 + 版本号
  - 原始语音与文本留链下（可加密），遵循授权与隐私规范

## 6. 核心流程
1) 活动创建 → 配置调研问题 → 发布活动
2) 参与签到 → 录制语音调研 → 提交
3) AI 处理 → 生成文本/情感/质量信号
4) 聚合计算 PeopleRank → 生成统计与置信度
5) 产出链上凭证（活动、个人维度）→ 前端展示
6) 机构、个人声誉随活动与反馈持续累积

## 7. 数据与算法（V0 草案）
- 单条反馈有效分
  - 内容相关度：回答围绕活动与问题（跑题降权）
  - 情绪/满意度：语音与文本抽取（作为一维，不是唯一信号）
  - 可信度惩罚：重复文本、极短回答、异常集中评分、批量新号等降权
- 活动聚合（PeopleRank）
  - 稳健统计：中位数/截尾均值，减少极端刷分影响
  - 置信度提示：样本数不足时不“封神”，提示“需要更多反馈”
  - 权重（MVP 可轻量）：参与证明 > 纯围观；高信誉反馈者略增权
- Global Reputation（跨社区）
  - 图结构：活动—机构—个人—反馈
  - 类 PageRank 传播：高质量活动提升机构；高信誉机构的活动有初始信任，但最终以 PeopleRank 校正

## 8. 激励与协作闭环
- 赞助/合作社区可为活动设置“奖励池”
  - 触达交换：打标/短分享/联合活动
  - 发放条件：PeopleRank 达到阈值 + 足够有效反馈
- 声誉与金钱分离
  - 声誉分不直接等同于金钱，避免“买分”
  - 权益发放使用声誉作为门槛或权重，更抗操纵

## 9. 风险与风控（MVP 必须）
- 女巫攻击（多号刷评）
  - 低成本门槛：签到码、最小时长、同设备/同网络限速
  - 可信度模型：相似度/聚类/时间分布异常
- 合谋互刷（组织内互捧）
  - 展示分布与样本结构：参与者/受众占比、重复率
  - 独立展示“可疑度”与“置信度”
- 隐私与合规
  - 默认不公开原始语音；仅公开摘要与统计
  - 存语音需加密 + 明确授权；遵循地区合规要求

## 10. 技术实现概要（黑客松方案）
- 前端：移动端优先的录音与提交界面；活动/机构/个人仪表盘
- AI 服务：ASR、NLP 摘要、情感分析、质量检测（可用现成 API 或轻量模型）
- 后端：活动/反馈/用户/机构的基础表；聚合分与统计计算任务
- 链上：活动结果摘要哈希与引用；参与凭证的 SBT/NFT（MVP 可先用 Attestation）
- 存储：链下安全存储文本与语音（可加密）；内容地址引用（如 IPFS/对象存储）

## 11. 里程碑与路演脚本
- 里程碑
  - D1：活动创建与语音提交跑通
  - D2：AI 分析与 PeopleRank 聚合展示
  - D3：链上绑定摘要 + 演示数据 + 路演稿
- 路演 60–90 秒脚本
  - 我们解决的是：NFT/SBT 发得多但不值钱的问题
  - 活动结束后，每人用 3 分钟语音回答 5 个问题
  - AI 把语音变结构化反馈：主题、满意度、要点、可疑度
  - 系统聚合成 PeopleRank，并把结果摘要哈希绑定到链上
  - 机构、个人声誉随真实反馈积累，赞助按 PeopleRank 触发发放

## 12. 后续迭代方向
- 更细粒度的维度分与权重学习
- 反馈者信誉的跨社区迁移与抗操纵优化
- 可验证的隐私计算（如可信执行环境/零知识摘要）
- 与更多合作社区的奖励池联动与策略测试
